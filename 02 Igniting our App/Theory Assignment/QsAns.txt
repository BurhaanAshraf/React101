Q: What is npm?
A: npm is the package manager for JavaScript/Node.js used to install, manage, and publish libraries.

Q: What is Parcel/Webpack? Why do we need them?
A: They are bundlers that bundle and optimize JS, CSS, and assets for development and production builds, improving performance and DX.

Q: Why is .parcel-cache created?
A: It stores build cache to speed up rebuilds by avoiding full recompilation.

Q: What is npx?
A: npx runs npm packages directly without global installation.

Q: What is the difference between dependencies and devDependencies?
A: dependencies are required in production; devDependencies are only needed for development/build.

Q: What is Tree Shaking?
A: Removal of unused code from the production bundle to reduce size.

Q: What is Hot Module Replacement (HMR)?
A: Updates changed modules in the browser without full page reload.

Q: List 5 superpowers of Parcel.
A: Dev server, HMR, Tree shaking, Minification, Caching.

Q: What is .gitignore? What should we add/not add?
A: It tells Git which files to ignore. Add node_modules, dist, .parcel-cache, .env. Do not add source code or package.json.

Q: Difference between package.json and package-lock.json?
A: package.json defines dependency ranges and scripts; package-lock.json locks exact installed versions.

Q: Why should you not modify package-lock.json?
A: It’s auto-generated to ensure consistent dependency versions; manual edits can break installs/builds.

Q: What is node_modules? Should it be pushed to Git?
A: Folder of installed dependencies; do not push because it’s large and regenerable.

Q: What is the dist folder?
A: Contains the final production build output generated by the bundler.

Q: What is browserslist?
A: Config that defines target browsers for transpilation and CSS prefixing.

Q: What do ^ and ~ mean in versions?
A: ^ allows minor+patch updates; ~ allows patch updates only.

Q: What does type="module" do in HTML?
A: Enables ES module syntax (import/export) in browser scripts.
